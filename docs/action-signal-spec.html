Formal signal definition

Action Signal Format

The app records one binary action signal for each room. These signals mark when the user is performing the instructed room action.

App Recording Spec Back Home

Signal Definition

For a home with N rooms, the app records N binary signals:

A(t) = [a_1(t), a_2(t), ..., a_N(t)]

For each room R_i:

  • a_i(t) = 1 only during the instructed action in room R_i.
  • a_i(t) = 0 otherwise.
  • All room signals are 0 during idle periods.
  • At most one room should be active in normal operation.

Action Start and End

Method-specific action interval definitions
Method Start End
AR + Button User taps Start Action before entering the target room. User taps Action Finished after exiting the target room.
QR Code User scans the QR code mapped to the target room before entering. User scans the same QR code again after exiting.
NFC Tag User taps the NFC tag mapped to the target room before entering. User taps the same NFC tag again after exiting.

QR/NFC Mapping

QR and NFC methods do not require a geometric room map. The app already knows valid QR-code IDs and NFC-tag IDs, and the user creates the room mapping during setup: 

room name <-> tag ID

Validation Rules

  • Only one room can be active at a time.
  • The same QR/NFC tag starts and ends the action.
  • The app rejects the wrong room tag.
  • Report Mistake restarts the affected step.
