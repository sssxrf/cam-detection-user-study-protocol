Formal signal definition

Action Signal Format

The app records one binary action signal for each area. These signals mark when the user is performing the instructed target-area action.

App Recording Spec Back Home

Signal Definition

For a home with N areas, the app records N binary action signals:

A(t) = [a_1(t), a_2(t), ..., a_N(t)]

For each area i:

  • a_i(t) = 1 only during intentional action in area i.
  • a_i(t) = 0 otherwise.
  • All area signals are 0 during idle periods.
  • At most one target-area action should be active in normal operation.
  • Transit should not be silently counted as a target-area action.

Action Start and End

Method-specific action interval definitions
Method Start End Transit handling
AR + Button User taps Start Action before entering the target area. User taps Action Finished after exiting the target area. The app logs required pass-through movement or keeps it outside the target-area action window.
QR Code User scans the QR code mapped to the target area before entering. User scans the same QR code again after exiting. The app prompts for any required transit and records it separately when needed.
NFC Tag User taps the NFC tag mapped to the target area before entering. User taps the same NFC tag again after exiting. The app prompts for any required transit and records it separately when needed.

QR/NFC Mapping

QR and NFC methods do not require a geometric room map. The app already knows valid QR-code IDs and NFC-tag IDs, and the user creates the area mapping during setup: 

area name <-> tag ID

Transit Options

During transit, the app uses one of two explicit options:

  • Option A: all area action signals remain 0 and a separate transit event is logged.
  • Option B: a transit signal for the pass-through area is recorded.

Important

Guard/transit areas are not treated as clean non-target areas during the current round.

Validation Rules

  • Only one target-area action can be active at a time.
  • The same QR/NFC tag starts and ends the action.
  • The app rejects the wrong area tag.
  • Transit events are logged separately or represented as transit signals.
  • Report Mistake restarts the affected step.
