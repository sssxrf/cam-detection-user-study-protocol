Multi-Room Hidden-Camera Detection Protocol

This site describes the user-facing protocol for a multi-room hidden-camera detection study. The user follows instructions from a mobile app while the app records room-level action signals and Wi-Fi metadata from a connected sniffer. For the AR + button method, the app assumes a room map is available on the phone. For the QR-code and NFC-tag methods, no room map is required; instead, the user first matches each room to a QR code or NFC tag. When the user taps Start Detection, the app starts the detection session and begins network capture through the connected sniffer.

Overview

The study app guides the user through room actions: stand outside a room, mark the start of the action, enter, walk naturally, exit, and mark the end. The app compares those action intervals with Wi-Fi metadata collected through the connected sniffer during the same session.

The goal is to study whether motion-activated Wi-Fi monitoring devices show traffic changes that repeatedly line up with room activity. The protocol does not prove that a device is a camera, and it should not be treated as a safety guarantee.

Participant instruction

The user should follow the app prompts, move naturally, and press Report Mistake if a scan, tap, room entry, or button press goes wrong.

What the App Records

The app records the timing of room actions and Wi-Fi metadata from the connected sniffer. It does not need participant names, exact home addresses, real MAC addresses, camera serial numbers, audio, or video for this protocol.

Choose a Method

The user study compares three ways for the app to record the room action interval.

AR + click button

The app has or loads a room map on the phone. The user confirms the rooms, then taps Start Action before entering and Action Finished after exiting.

QR code

No geometric room map is required. The user places or selects one QR code at each room entrance, scans it, names the room, and confirms the mapping.

NFC tag

No geometric room map is required. The user places or selects one NFC tag at each room entrance, taps it, names the room, and confirms the mapping.

Room Setup

Room setup for QR and NFC methods

In the QR-code and NFC-tag methods, the user performs the room setup. The app already recognizes the available QR-code or NFC-tag IDs, but it does not know which physical room each tag represents. The user places or selects one tag at each room entrance, scans or taps it, enters the room name, and confirms the mapping. After setup, the app can guide the user room by room without needing a geometric room map.

  1. Choose the number of rooms. The user tells the app how many rooms will be included in the study.
  2. Assign a room tag. For each room, the user places or selects one QR code or NFC tag near the room entrance.
  3. Register the tag. The user scans the QR code or taps the NFC tag.
  4. Name the room. The user enters a room name, such as Bedroom, Kitchen, Living Room, Office, or Bathroom.
  5. Confirm the mapping. The app shows the tag ID and room name. The user confirms that the mapping is correct.
  6. Review all rooms. The app shows a verification table.
  7. Start detection. After all rooms are confirmed, the user taps Start Detection. At that moment, the app starts the session and starts network capture through the connected sniffer.

Start Detection

After room setup is complete, the user taps Start Detection. This single action starts the guided detection session and starts network capture through the connected sniffer. The user does not separately operate the sniffer. From this point on, the app controls the timing of room actions and idle periods.

  1. User opens the app.
  2. User chooses the method: AR, QR, or NFC.
  3. User completes any required room setup.
  4. User taps Start Detection.
  5. The app starts the session timer, initializes the connected sniffer/network capture, and begins guiding room actions.

AR + Button Instructions

Participant instruction

  1. Confirm the room list and starting location shown by the app.
  2. Tap Start Detection.
  3. Stand outside the room shown in the app.
  4. Tap Start Action.
  5. Enter the room naturally.
  6. Walk along the instructed path.
  7. Exit the room.
  8. Tap Action Finished.
  9. Wait for the next prompt.

The app records the interval between Start Action and Action Finished as action=1 for the target room. All other room action signals remain 0.

QR Code Instructions

When the app asks the user to test a room, the user scans that room's tag before entering. This marks the start of the action interval for that room. After walking naturally inside the room and exiting, the user scans the same tag again. This marks the end of the action interval. The app sets that room's action signal to 1 between the two scans and 0 otherwise. All other room action signals remain 0.

  1. During setup, place or select one QR code near each room entrance.
  2. Scan the QR code, name the room, and confirm the mapping.
  3. Tap Start Detection after all rooms are confirmed.
  4. When prompted, stand outside the target room.
  5. Scan the assigned QR code before entering.
  6. Enter, walk naturally, and exit.
  7. Scan the same QR code again after exiting.
  8. Wait for the next prompt.

NFC Tag Instructions

When the app asks the user to test a room, the user taps that room's tag before entering. This marks the start of the action interval for that room. After walking naturally inside the room and exiting, the user taps the same tag again. This marks the end of the action interval. The app sets that room's action signal to 1 between the two taps and 0 otherwise. All other room action signals remain 0.

  1. During setup, place or select one NFC tag near each room entrance.
  2. Tap the NFC tag, name the room, and confirm the mapping.
  3. Tap Start Detection after all rooms are confirmed.
  4. When prompted, stand outside the target room.
  5. Tap the assigned NFC tag before entering.
  6. Enter, walk naturally, and exit.
  7. Tap the same NFC tag again after exiting.
  8. Wait for the next prompt.

Error Handling

  • If the user scans or taps the wrong room tag, the app should show an error.
  • If the app asks for Bedroom but the user scans Kitchen, the app should say: "This tag is registered to Kitchen, but the current room is Bedroom. Please scan the Bedroom tag or press Report Mistake."
  • If the user forgets a scan or tap, they should press Report Mistake.
  • If an action is currently active for one room, the app should not allow another room action to start.
  • The app should restart the affected step after a mistake.
  • Mistakes include wrong room, wrong scan or tap, forgotten scan or tap, failed QR scan, failed NFC tap, entering too early, or exiting late.

Multi-Room Scheduling

Sequential mode

  • The app tests one room at a time.
  • The app keeps the room idle for the reset or cooldown period.
  • The app guides four room actions separated by idle intervals.
  • The app moves to the next room after the scheduled actions.
  • This is slower but cleaner.

Interleaved mode

  • The app may use idle time for one room to test another room.
  • The app should never schedule overlapping actions.
  • The user should not enter a room during its required idle period.
  • The app should keep a short separation between actions in different rooms.
  • The app should avoid identical periodic schedules across rooms.
  • This is faster but requires careful app timing.

Room state machine

Room state ∈ {SETUP, RESETTING, READY_FOR_ACTION, ACTION_ACTIVE, WAITING_IDLE, CLASSIFIED, NEEDS_DISAMBIGUATION}

Action Signal Format

For a home with N rooms, the app records N binary action signals: 

A(t) = [a_1(t), a_2(t), ..., a_N(t)]

For each room R_i:

  • a_i(t) = 1 only while the user is performing the instructed enter-walk-exit action in room R_i.
  • a_i(t) = 0 otherwise.

During normal operation, only one room should have signal 1 at a time. During idle periods, all room signals are 0.

Privacy and Safety

Privacy caution

  • The app records action timestamps and Wi-Fi metadata.
  • The app should not record audio or video unless explicitly stated.
  • Device identifiers should be pseudonymized before analysis.
  • The user may stop at any time.
  • The user should not use the protocol in unsafe situations.
  • The public site should not include real addresses, participant names, real MAC addresses, or sensitive layouts.
