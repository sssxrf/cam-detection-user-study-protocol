App Recording Spec

This page explains what the app records during the guided session and what information is not needed for the public protocol.

Participant Instructions Back Home

Start Trigger

After room setup is complete, the user taps Start Detection. This starts both the guided session and network capture through the connected sniffer. The user does not separately start or operate the sniffer.

What the App Records

Session records
Category Examples
Session metadata Random participant ID, random session ID, selected method, app version, session start and end time.
Room setup mappings Room number, room name, QR/NFC tag ID when applicable, confirmation status.
Action events Room ID, action start time, action end time, action duration, idle duration, retry count.
QR/NFC events Tag ID, scan/tap timestamp, expected room, scanned/tapped room, success or failure.
AR/button events Start Action timestamp, Action Finished timestamp, AR tracking state, tracking failure events.
Wi-Fi metadata Capture start/end time, pseudonymized device ID, packet timestamp, packet size, frame type, RSSI, and channel when available.
Mistake reports Report Mistake timestamp, affected room or step, retry count, and the app's restart state.

What Is Not Needed

  • Participant names.
  • Exact addresses.
  • Real MAC addresses in public outputs.
  • Audio or video unless separately stated.
  • Camera serial numbers.
  • Sensitive floor plans in public materials.

App Responsibilities

  • Maintain one active room action at a time.
  • Reject mismatched tags.
  • Restart affected steps after Report Mistake.
  • Stop capture when the session ends.
