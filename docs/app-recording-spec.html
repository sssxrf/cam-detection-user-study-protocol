App Recording Spec

This page explains what the app records during area setup, connection setup, grouped detection rounds, and the guided session.

Start Trigger

After area setup, connection setup, and app-generated group review are complete, the user taps Start Detection. This starts both the guided session and network capture through the connected sniffer. The user does not separately start or operate the sniffer.

What the App Records

Session records
Category Examples
Session metadata Random participant ID, random session ID, selected method, app version, session start and end time.
Area setup mappings Area number, area name, QR/NFC tag ID when applicable, confirmation status.
Connection setup Areas, direct connections, visibility links, doorway/open-boundary links, pass-through routes, confirmed connection graph.
Automatic grouping Conflict graph inputs, app-generated groups, round order, target group for each round, guard/transit areas, idle non-target areas.
Action events Area ID, action start time, action end time, action duration, idle duration, transit event when used, retry count.
QR/NFC events Tag ID, scan/tap timestamp, expected area, scanned/tapped area, success or failure.
AR/button events Start Action timestamp, Action Finished timestamp, AR tracking state, area entry/exit estimate when available, tracking failure events.
Wi-Fi metadata Capture start/end time, pseudonymized device ID, packet timestamp, packet size, frame type, RSSI, and channel when available.
Mistake reports Report Mistake timestamp, affected area, route, or step, retry count, and the app's restart state.
Disambiguation outcomes Ambiguous device marker, extra-test type, and labels such as area-specific, shared-view, hallway/transit, or unresolved candidate.

What Is Not Needed

  • Participant names.
  • Exact addresses.
  • Real MAC addresses in public outputs.
  • Audio or video unless separately stated.
  • Camera serial numbers.
  • Sensitive floor plans in public materials.

App Responsibilities

  • Maintain one active target-area action at a time.
  • Build a conflict graph from direct connections, visibility, doorway conflicts, and pass-through relationships.
  • Use graph coloring to split areas into two or more non-conflicting groups.
  • Run one detection round per group and make every area a target area in one round.
  • Treat adjacent, visible, and pass-through areas as guard/transit areas instead of clean non-target evidence.
  • Log transit separately or record a transit signal for the pass-through area.
  • Reject mismatched tags.
  • Restart affected steps after Report Mistake.
  • Stop capture when the session ends.
